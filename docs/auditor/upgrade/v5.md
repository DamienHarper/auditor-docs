# Upgrading to v5.0

## Breaking Changes

### DoctrineProvider removed from core

The `DoctrineProvider` and all classes under `DH\Auditor\Provider\Doctrine\` have been **removed** from `damienharper/auditor`.

They are now available in the standalone package `damienharper/auditor-doctrine-provider`.

**Migration steps:**

1. Install the new package:
   ```bash
   composer require damienharper/auditor-doctrine-provider
   ```

2. No code changes needed — the namespace `DH\Auditor\Provider\Doctrine\` is preserved.

3. If you use `auditor-bundle`, it already requires `auditor-doctrine-provider` — no action needed.

### Removed dependencies

`damienharper/auditor` v5.0 no longer requires:
- `doctrine/dbal`
- `doctrine/orm`
- `symfony/cache`
- `symfony/lock`

These are now dependencies of `damienharper/auditor-doctrine-provider`.
